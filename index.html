💡 3 padrões de UX pra lançamento seguro (escolha um)

Como evitar lançar valor por engano?

Clique único é perigoso (errou e gravou no banco). Aqui estão 3 opções de UX, cada uma com um "atrito" diferente. Você vai usar isso toda hora — peso da escolha é alto.

A) Clique → input → Salvar ⭐ recomendado

1 clique na célula abre input. Você digita. Clica em ✓ Salvar (ou Enter). Sem botão Salvar não grava nada — fechar a célula clica fora cancela.

↑ é assim que aparece no clique
✓ 2 passos só: rápido pra uso diário
✓ Impossível salvar sem confirmar
✓ Esc / clique-fora = cancela
✕ Pode passar batido se você confirma no automático
B) Clique → input → Modal "tem certeza?"

Mesmo fluxo de A, mas depois do Salvar abre modal mostrando valor antigo → novo. Você precisa confirmar de novo.

⚠ Confirmar alteração
R$ 80.000 → R$ 62.000
✓ Mostra valor velho × novo — pega erros de zero
✓ Pega audit trail "pq mudou"
✕ 3 cliques pra lançar — fica chato em uso intensivo
✕ Pode ativar só pra mudança grande (>30% delta)
C) Botão "Fechar mês" → formulário em lote

Sem edição inline. Você clica "Fechar Maio/26" e abre form com TODOS indicadores do setor pra preencher de uma vez. Cada campo tem label, valor atual, novo valor.

Fechar Maio/26 · Comercial
Novas vendas (meta: 160)
Faturamento (meta: R$ 80k)
+ 6 indicadores
✓ Foco em fechamento ritualizado (1×/mês)
✓ Vê tudo do mês de uma vez — contexto pleno
✕ Não dá pra corrigir 1 célula isolada
✕ Fluxo único: precisa "fechar" o mês inteiro
💡 Minha recomendação: Opção A (default) + Opção B ativada automaticamente quando o delta é grande (>30% pra mais OU pra menos). Assim 95% do uso é rápido (A) e os 5% perigosos pegam confirmação extra (B). Opção C pode entrar como botão "Fechar mês" no canto pra quando você quer ritualizar.
